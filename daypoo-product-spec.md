> 작성일: 2026-07-10
> 목적: gstack 기반 신규 프로젝트 구성을 위한 서비스 기획 참고 문서
> 기술 스택 내용 제외 — 순수 서비스 목적 및 기능 정의에 집중

---

## 1. 서비스 개요

### 서비스명
**DayPoo** — *"당신의 흔적이 건강이 됩니다"*

### 한 줄 소개
대한민국 전국 화장실 정보를 지도 위에 시각화하고, 사용자의 배변 기록을 AI로 분석하여 생활 습관 인사이트를 제공하는 **위치 기반 헬스케어 서비스**.

### 탄생 배경

- 급한 상황에서 주변 화장실을 빠르게 찾을 수 있는 실용적인 서비스의 필요성
- 배변 습관이 건강의 중요한 지표임에도 불구하고, 이를 체계적으로 기록하고 분석해주는 서비스가 부재
- 공공 화장실 데이터(전국 약 5만 건)가 공개되어 있으나 사용자 친화적으로 제공되지 않고 있음

---

## 2. 서비스 목적 및 핵심 가치

### 핵심 가치 (4축)

| 가치 | 설명 |
|------|------|
| **급똥 대응** | 위치 기반으로 가장 가까운 화장실을 즉시 안내 |
| **방문 인증 & 기록** | 실제 방문한 화장실에 GPS 체크인하여 개인 배변 이력 누적 |
| **AI 건강 분석** | 배변 사진 및 상태 태그 기반으로 건강 인사이트 제공 |
| **게이미피케이션** | 방문 기록이 경험치·포인트·랭킹으로 이어지는 동기부여 시스템 |

### 목표 사용자

- 화장실 위치 정보가 절실한 외출 중 사용자 (급똥 대응)
- 자신의 장 건강·배변 습관을 데이터로 관리하고 싶은 사용자
- 배변 기록을 통한 생활 습관 개선에 관심 있는 건강 관리자

### Privacy-First 설계 철학

> 배변 이미지는 극도로 민감한 개인 정보.
> AI 분석에는 이미지를 활용하지만, **서버 DB에는 절대 저장하지 않고** 분석 완료 즉시 메모리에서 폐기.
> 사용자에게는 분석 결과(텍스트/수치)만 저장·표시됨.

## 3. 주요 기능 목록

### 3.1 지도 & 화장실 탐색

| 기능 | 설명 |
|------|------|
| 전국 화장실 지도 표시 | 전국 공공 화장실 약 5만 건을 지도 위에 마커로 시각화 |
| 마커 상태 구분 | 미방문(회색) / 방문 완료(컬러) 2단계 마커 시스템으로 정복감 유도 |
| 급똥 버튼 | 현재 위치 기준 500m~1km 반경 화장실 즉시 TOP 3 추천 (운영 시간 가중치 적용) |
| 화장실 상세 정보 | 이름, 주소, 운영 시간, 남녀공용 여부 등 확인 가능 |
| 주변 화장실 검색 | 위치 기반으로 근처 화장실 목록 조회 및 거리 순 정렬 |

### 3.2 방문 인증 & 배변 기록

| 기능 | 설명 |
|------|------|
| GPS 체크인 | 화장실 50m 반경 도착 시 체크인 가능 (서버에서 좌표 검증) |
| 체류 시간 검증 | 체크인 후 1분 이상 체류해야 배변 기록 작성 가능 (허위 인증 방지) |
| 배변 기록 작성 | 브리스톨 척도(1~7단계), 색상, 컨디션 태그, 식이 태그, 경고 태그 등 입력 |
| 이미지 AI 분석 | 배변 사진 촬영 → AI가 자동 분석하여 브리스톨 척도 및 이상 징후 감지 |
| 방문 완료 처리 | GPS 재검증 후 방문 완료 처리 및 마커 색상 변경 |

### 3.3 AI 건강 분석 & 리포트

| 기능 | 설명 |
|------|------|
| 실시간 AI 분석 | 배변 기록 생성 시 이미지를 AI가 즉시 분석 (브리스톨 척도, 이상 감지) |
| 주간/월간 건강 리포트 | 누적 배변 기록 기반 통계 + AI 코멘트 생성 |
| 구독 등급별 리포트 범위 차등 | FREE는 기본 통계, PREMIUM은 상세 AI 분석 리포트 제공 |

### 3.4 게이미피케이션

| 기능 | 설명 |
|------|------|
| 경험치(EXP) & 레벨 | 방문 인증·배변 기록 시 경험치 획득 및 레벨업 |
| 포인트(Point) | 활동 시 포인트 획득 → 상점에서 아이템 구매에 사용 |
| 글로벌 랭킹 | 전체 사용자 기준 실시간 랭킹 집계 |
| 지역별 랭킹 | 지역 단위 랭킹 집계 (지역 커뮤니티 경쟁 요소) |
| 칭호 시스템 | 누적 활동량 기반 칭호 획득 (예: "변기 정복자", "장 건강 마스터") |

## 4. 핵심 비즈니스 규칙 (Business Rules)

### 4.1 방문 인증 규칙

- 체크인 가능 거리: 화장실로부터 **50m 이내**
- 유효 방문 기준: 체크인 후 **1분 이상** 체류
- 기록 저장 시점: 체류 확인 후 GPS 재검증 완료 시점
- 이벤트 유형: CHECK_IN / COMPLETE / ABANDON

### 4.2 AI 분석 이미지 처리 규칙

- 이미지는 AI 분석 목적으로만 전송
- 분석 즉시 서버 메모리에서 폐기 (DB 저장 없음)
- 저장되는 데이터: 브리스톨 척도, 색상, 경고 태그 등 분석 결과값만

### 4.3 구독 플랜 차등 규칙

| 기능 | FREE | PREMIUM |
|------|------|---------|
| 배변 기록 | O | O |
| 기본 통계 | O | O |
| AI 리포트 (상세) | X | O |
| 리포트 기간 | 주간 | 주간 + 월간 |

### 4.4 랭킹 집계 규칙

- 글로벌 랭킹: 전체 유저 대상 실시간 집계
- 지역 랭킹: region_name 기반 지역 단위 분리 집계
- 갱신 시점: 배변 기록 저장 완료 즉시 실시간 반영

## 5. 데이터 모델 개요 (개념적 수준)

### 핵심 엔티티 관계

```
User (사용자)
 ├── PooRecord (배변 기록) — 1:N
 │     └── AI 분석 결과 포함 (이미지 비저장)
 ├── VisitLog (방문 로그) — 1:N
 │     └── Toilet (화장실) 참조
 ├── Subscription (구독 정보) — 1:1
 ├── Title (칭호) — N:1 (장착 칭호)
 └── Item (보유 아이템) — N:M

Toilet (화장실)
 ├── 위치 정보 (위도/경도)
 ├── 운영 시간
 └── VisitLog 참조

Item (상점 아이템)
 └── 카테고리: 아바타 / 마커 스킨
```

### 배변 기록 주요 속성

- **브리스톨 척도**: 1~7 (변의 형태 분류 국제 기준)
- **색상**: 배변 색상 기록
- **컨디션 태그**: 사용자가 선택하는 신체 컨디션 태그
- **식이 태그**: 식사 관련 태그 (음주, 자극적 음식 등)
- **경고 태그**: 혈변, 통증 등 이상 증상 태그
- **지역명**: 방문 화장실의 지역 분류

## 6. 사용자 시나리오 (User Journey)

### 시나리오 A: 급똥 대응

1. 외출 중 급한 상황 발생
2. 앱 실행 → 지도 화면 진입
3. **급똥 버튼** 탭
4. 현재 위치 기준 TOP 3 화장실 추천 표시 (거리 + 운영 중 여부)
5. 화장실 선택 → 길 안내 이동

### 시나리오 B: 방문 인증 & 기록 작성

1. 화장실 50m 이내 도착
2. 앱에서 해당 화장실 마커 탭 → **체크인** 버튼 클릭
3. 볼일 완료 후 **기록 작성** 버튼 클릭 (1분 이상 체류 확인)
4. 브리스톨 척도, 컨디션 태그 등 입력
5. (선택) 사진 촬영 → AI 자동 분석
6. 저장 완료 → 경험치·포인트 지급, 마커 색상 변경(회색 → 컬러)
7. 랭킹 즉시 업데이트

### 시나리오 C: 건강 리포트 확인

1. 마이페이지 → 건강 리포트 탭 진입
2. 주간 배변 횟수, 브리스톨 척도 분포, 경고 태그 빈도 확인
3. AI 코멘트 확인 ("최근 1주일 배변 패턴이 불규칙합니다. 식이섬유 섭취를 늘려보세요.")
4. (PREMIUM) 월간 트렌드 분석 및 상세 건강 리포트 확인

## 7. 확장 가능성 (Future Features)

- **커뮤니티 기능**: 화장실 리뷰·별점 기능 (방문 인증 완료 사용자만)
